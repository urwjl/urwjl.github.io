Intel DSA 相关

发表于 2024-10-08 更新于 2024-11-15 分类于 intel ， DSA ， 指令
收集了一些Intel DSA 相关信息

Intel DSA 相关

Intel DSA 相关指令

  • ENQCMD 、ENQCMDS 是Non-Posted，需要Completer 返回状态，用于共享的工作队列
  • MOVDIR64B 是Posted，不需要返回状态，用于专用的工作队列
  • UMONITOR 、UMWAIT 实现更高效的等待和监控功能
  • SFENCE 、 MFENCE 内存屏障，在写指令之后插入写屏障，能让写入缓存的最新数据写回到主内存

提交工作描述符

ENQCMD / ENQCMDS

Shared Virtual Addressing (SVA) with ENQCMD

在 Intel CPU 上，DMWr 是使用 ENQCMD 或 ENQCMDS 指令生成的。ENQCMD 和 ENQCMDS 指令在 EFLAGS.ZF 标志中返回命令提交的状态；0 表示成功，1 表示重试。ENQCMD和 ENQCMDS 也自动向设备提交 64 字节工作描述符。但是，它们可以直接从设备报告错误，例如设备是否繁忙、设备未启用或不支持该命令。提交指令本身的这种即时反馈减少了与设备的交互次数，并可以大大提高效率。

DMWr (Deferrable Memory Write)是一种新的 PCIe TLP 类型。CXL 1.1 中就已经有 Deferrable Writes 了，正式出现在 PCIe 协议中是 PCIe 6.0。

可延迟的内存写入，顾名思义，Completer 在收到 DMWr 后，可以选择立即执行，也可以选择延迟执行。相比于转发请求 MWr，DMWr 是非转发请求（NPR），无论请求是否被执行，Completer 都应根据完成情况回复指定状态的 Completion。

ENQCMD 原子地将 64 字节用户命令从源内存操作数排入 ES 段中的目标偏移量（在寄存器操作数中指定为 ES 段中的偏移量）。

位 19:0 传达进程地址空间标识符 (PASID)，系统软件可能会将这个值分配给各个软件线程。位 31 包含权限标识（0 = 用户；1 = 管理员）。实现排队寄存器的设备可以将这两个值与高 60 个字节中的设备特定命令一起使用。

ENQCMD 指令首先从其源内存操作数读取 64 个字节的命令数据。这是一个普通的加载，具有缓存能力和内存排序，通常由内存类型暗示。源操作数不需要对齐，并且不能保证所有 64 个字节都自动加载。源操作数的位 31:0 必须为零。

然后，该指令将这 64 个字节格式化为命令数据，其格式与图 1-16 中给出的格式一致：

  • • Command[19:0] 获取 IA32_PASID[19:0].
  • • Command[30:20] 为 0。
  • • Command[31] 为 0（表示用户；无论 CPL 如何，都会使用此值）。
  • • Command[511:32] 获取从内存读取的源操作数的位 511:32。

ENQCMD 指令使用排队存储（定义如下）将此命令数据写入目标操作数。目标操作数的地址在通用寄存器中指定为 ES 段的偏移量（该段不能被覆盖）。 目标线性地址必须是 64 字节对齐的。排队存储的操作忽略目标内存地址的内存类型。

ENQCMD 和 ENQCMDS 指令各自执行 64 字节入队存储，其中包括 19:0 位中的 20 位 PASID 值。

对于 ENQCMD，PASID 通常是 IA32_PASID[19:0] 的值，而对于 ENQCMDS，PASID 通常是从内存中读取的。

PASID 转换

如果“PASID 转换”VM 执行控制为 1，则上一段中标识的 PASID 值将被视为客户机 PASID。PASID 转换将此客户机 PASID 转换为 20 位主机 PASID。在此转换之后，将执行入队存储，使用主机 PASID 代替客户机 PASID。

PASID 转换由 VMM 配置的两个数据结构层次结构（PASID 转换层次结构）实现。客户机 PASID 00000H 至 7FFFFH 通过低 PASID 转换层次结构进行转换，而客户机 PASID 80000 至 FFFFFH 通过高 PASID 转换层次结构进行转换。

每个 PASID 转换层次结构的根都是一个 4 KB 的 PASID 目录。低 PASID 目录位于低 PASID 目录地址，高 PASID 目录位于高 PASID 目录地址（这些物理地址是 VMCS 中的 VM 执行控制字段）。PASID 目录包含 512 个 8 字节条目，每个条目的格式如下：

  • • 位 0 是条目的当前位。仅当此位为 1 时才使用该条目。
  • • 位 11:1 是保留位，必须为 0。
  • • 位 M–1:12 指定 PASID 表的 4 KB 对齐地址（见下文），其中 M 是处理器的物理地址宽度。
  • • 位 63:M 是保留位，必须为 0。

PASID 转换层次结构还包括最多 512 个 4 KB PASID 表；每个表都由 PASID 目录条目引用（见上文）。PASID 表包含 1024 个 4 字节条目，每个条目的格式如下：

  • • 位 19:0 是条目指定的主机 PASID。
  • • 位 30:20 是保留位，必须为 0。
  • • 位 31 是条目的有效位。仅当此位为 1 时才使用该条目。

当启用 PASID 转换时，由指令（参见上文）确定的客户机 PASID 将使用以下过程转换为主机 PASID：

  • • 如果客户机 PASID 的第 19 位已清除，则使用低 PASID 目录；否则，使用高 PASID 目录。
  • • 客户机 PASID 的第 18:10 位从 PASID 目录中选择一个条目。如果条目的当前位已清除或设置了任何保留位，则发生 VM 退出。否则，条目的位 M:0（位 0 已清除）包含 PASID 表的物理地址。
  • • 客户机 PASID 的第 9:0 位从 PASID 表中选择一个条目。如果条目的有效位已清除或设置了任何保留位，则发生 VM 退出。否则，条目的位 19:0 为主机 PASID。

如果 PASID 转换导致 VM 退出（由于现有或有效位被清除，或者保留位被设置），则该指令不会完成并且不会执行任何入队存储。

MOVDIR64B

在 Intel CPU 上，使用 MOVDIR64B 指令将工作提交到 DWQ，该指令生成非撕裂（non-torn）的 64 字节写入。==如果使用 MOVDIR64B 写入已禁用的 WQ、共享的 WQ 或无效的通道地址，则该写入将被丢弃，而不会通知软件。==对于专用 WQ，PASID 的使用是可选的。如果未启用 PCI Express PASID 功能，则不使用 PASID。如果启用了 PASID 功能，WQ 配置寄存器的 WQ PASID 启用字段控制是否为每个 DWQ 使用 PASID。由于 MOVDIR64B 指令不会像 ENQCMD 或 ENQCMDS 指令那样填写 PASID，因此描述符中的 PASID 字段将被忽略。当为 DWQ 启用 PASID 时，设备使用 WQ 配置寄存器的 WQ PASID 字段进行地址转换。在专用模式下启用工作队列之前，驱动程序必须设置 WQ PASID 字段。

MOVDIR64B只能向专用工作队列提交描述符，不能向共享工作队列提交描述符，会被丢弃

目前，MOVDIR64B 指令用于自动向设备提交 64 字节工作描述符。虽然在写入设备 MMIO 时可能会遇到设备队列已满、命令未接受、设备未准备好等错误，但 MOVDIR64B 无法报告设备本身的错误。这意味着 MOVDIR64B 用户需要单独与设备交互以查看描述符是否已成功排队，这会减慢设备交互速度。

内存屏障

SFENCE / MFENCE

正在使用设备的进程可以在正常关机期间使用 Drain 描述符。它可以像整个 PASID 的 Fence 操作一样使用。它可用于请求单个完成记录和/或中断以完成多个描述符。批处理中不能包含 Drain 描述符。（批处理中可以使用 Fence 标志来等待批处理中的先前描述符完成。）软件应在提交 Drain 描述符之前执行隔离指令（例如 SFENCE 或 MFENCE），以确保设备在要Drain 的描述符之后收到 Drain 描述符。

完成记录地址

UMONITOR / UMWAIT

使用完成记录地址上的 UMONITOR/UMWAIT 指令进行阻塞，直到写入或超时。然后，软件应检查状态字段是否为非零，以确定操作是否已完成。

umonitor指令用于监控指定内存地址的变化，当监控的内存地址发生变化时，umonitor会触发一个事件，从而可以及时采取相应的处理措施。umonitor指令可以提高云计算和服务器应用中的事件处理效率，减少不必要的轮询操作，节省系统资源。

umwait指令用于等待指定事件的发生，可以在等待期间进入低功耗状态，从而降低能耗。umwait指令可以在云计算和服务器应用中实现更高效的等待和唤醒机制，提高系统的能效。

Intel Sapphire Rapids CPU，吹响反攻DPU的号角

AIA（Accelerator interfacing Architecture，加速器接口架构）不是一个简单的功能或特征实现，而是一组相关技术能力的组合（类比Intel的VT-x和VT-d技术）。从Intel新一代Xeon处理器Sapphire Rapids开始，提供加速接口技术AIA，其技术点包括如下：

  • 任务分配指令（MOVDIRI、MOVDIR64B、ENQCMD/S）用于优化任务卸载，ENQCMD/S支持共享任务队列；
  • 用户态等待指令（UMONITOR、UMWAIT、TPAUSE），用于高效同步；
  • 低延迟用户态中断；
  • 共享虚拟内存；
  • 轻量的可扩展I/O虚拟化S-IOV。

AIA目前支持的加速类型有：数据流处理DSA、加解密和数据压缩QAT等。
