ATS简介

发表于 2024-11-04 更新于 2024-11-15 分类于 PCIe ， ATS
地址转换服务 (ATS) 是外围组件互连 Express (PCIe) 6.0 架构中的关键进程

地址转换服务 (ATS) 是外围组件互连 Express (PCIe) 6.0 架构中的关键进程。它在确保系统上运行的不同进程或应用程序不会干扰彼此的内存空间方面发挥着关键作用。每个进程都在其虚拟地址空间内运行，随后将其转换为物理地址。此机制对于保持系统稳定性和防止内存冲突至关重要。

ATS 与 PCIe 6.0 中的进程请求标识符 (PRI) 和进程地址空间标识符 (PASID) 一起，可显著增强设备的功能。但是，这些增强功能也增加了创建和调试测试场景的复杂性。

验证策略应侧重于确保 PCIe 环境中增强的 ATS 功能的稳健性和可靠性。它涉及一种系统化的方法来测试不同的场景，包括功能测试、性能测试和压力测试。目标是在开发过程的早期识别和解决潜在问题，降低缺陷风险并确保流畅的用户体验。

本博客是系列博客的第一篇，介绍了 ATS 的要求以及 PCIe 协议代中提出的各种组件。本系列的后续博客将介绍 PCIe 6.0 中地址转换验证所涉及的挑战和策略的宝贵见解。

地址转换简介

地址转换是将虚拟地址（程序使用的逻辑地址）转换为物理地址（数据在内存中的实际位置）的过程。地址转换可以由两个不同的单元执行，它们维护转换映射：

  • 内存管理单元 (MMU) – 将 CPU 虚拟地址转换为物理地址
  • 输入/输出内存管理单元 (IOMMU) – 将设备 I/O 虚拟地址转换为物理地址

地址转换过程

地址转换过程如下：

  • 当程序引用内存位置时，虚拟地址将发送到 MMU。
  • MMU 检查其转换后备缓冲区 (TLB) 以查看虚拟地址是否最近被转换过。
  • 如果在 TLB 中找到虚拟地址，则将物理地址返回给处理器。
  • 如果在 TLB 中找不到虚拟地址，MMU 将执行页表遍历以查找物理地址。
  • 然后将物理地址返回给处理器。

地址转换的好处

MMU 和 IOMMU 分别记录由 CPU 和设备发起的地址转换。以下是地址转换的一些好处：

  • 内存保护：地址转换允许操作系统通过防止未经授权访问某些内存区域来保护内存。这对于确保系统及其数据的安全至关重要。
  • 虚拟内存：地址转换允许使用虚拟内存，从而使操作系统能够更有效地管理内存。虚拟内存允许操作系统通过使用硬盘的一部分作为当前未使用的数据的临时存储区域，创建比可用物理内存更大的假象。
  • 缓存一致性：地址转换允许操作系统保持缓存一致性，这是一种确保内存中数据项的所有副本都一致的技术。
  • 设备隔离：地址转换有助于将设备彼此隔离，从而防止冲突并提高系统稳定性。
  • 减少处理开销：为了减少处理器中执行的地址转换开销，引入了设备中的地址转换缓存 (ATC)，与 ATS 一起使用。ATC 在访问主机内存时提供地址转换映射的设备中维护。ATS 为端点 (EP) 提供了一种方法来维护与根复合体 (RC) 中维护的地址转换和保护表 (ATPT) 一致的 ATC。

ATS 组件

转换代理 (TA)，可以是 IOMMU 或 MMU，由 Hypervisor/Software 管理。它代表主机为 PCIe 设备转换地址，确保安全性并授予对主机内存的访问权限以进行 DMA 事务。TA 使用页表等数据结构并与 RC 通信以生成请求的转换 TLP 的成功或失败完成。

地址转换保护表 (ATPT) 是一种用于存储地址转换页表的数据结构。它包含由转换代理 (TA) 访问以处理 PCIe 请求的一组地址转换。DMA 读取和写入请求通过路由 ID 和 PCIe 事务中包含的地址的组合进行转换。

地址转换缓存 (ATC) 是设备或端点中的缓存，用于存储虚拟内存系统中虚拟地址和物理地址之间的转换。它充当缓存层，通过存储最近的虚拟到物理地址映射来加快转换过程。ATC 通过降低 CPU 的延迟来提高系统性能。每个 ATC 条目都必须通过 ATS 转换完成来填充，以响应发出 ATS 转换请求的功能。ATC 仅存在于物理功能中。

摘要

ATS 是 PCIe 标准中的一项功能，允许 I/O 设备执行自己的地址转换，而无需依赖主机的 CPU。这有助于减少延迟并提高整体系统性能。

支持 ATS 的设备将支持直接内存访问 (DMA) 的 I/O 总线连接到主内存。当启用 ATS 的设备需要访问内存时，它可以使用 ATS 功能将内存位置的虚拟地址转换为物理地址。此转换由硬件完成，比由 CPU 在软件中完成转换要快得多。

使用 ATS 有几个好处。首先，它可以减少延迟。当 I/O 设备需要访问内存时，它不必等待 CPU 执行地址转换。这可以提高需要快速访问内存的应用程序的性能。其次，ATS 可以提高整体系统性能。通过减少 CPU 上的负载，ATS 可以释放 CPU 来执行其他任务。这可以提高整个系统的性能。

地址转换是现代计算机系统的重要组成部分。它提供了几个好处，包括内存保护、虚拟内存和缓存一致性。
